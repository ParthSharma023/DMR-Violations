# data/ — Phase 4 pipeline output

All parquet files here are generated by `tools/build_data.py`. They are the canonical, validated snapshot of the PBIX data model.

To rebuild: `python tools/build_data.py` (takes ~60 s, reads 5.5 GB of CSVs from `source/exported_csv/`).

## tier2/ — cleaned base + calculated tables

Parquet (ZSTD-compressed). Total ~1.3 GB → ~180 MB compressed.

### Base tables (direct from CSV, with global filters applied)

Global filters on DATATBL and VARDESC:
- `WWTP NOT IN` 11-value exclusion list (Bretshire*, Eastheven, Northside*, Scott*, Willow Run, SUM*)
- `WWTP IS NOT NULL`
- `DATESTAMP >= 2005-01-01` (DATATBL only)

## Validation

Violation counts per WWTP match the archived v1 output exactly, with one expected correction:

- v1 summary included `Willow Run` (38 violations) — v1 omitted the Willow Run filter
- v2 excludes `Willow Run` per the PBIX report-level filter
- All other 40+ plants match to the exact violation count

Totals: v2 = 12,208 violations | v1 = 12,246 | diff = −38 (exactly Willow Run)

## Reading parquet

From Python:
```python
import pandas as pd
df = pd.read_parquet("build/tier2/DATATBL.parquet")
```

From DuckDB (faster for large queries):
```python
import duckdb
df = duckdb.sql("SELECT WWTP, SUM(Violation) FROM 'build/tier2/DATATBL.parquet' GROUP BY 1").df()
```

From the browser (via `parquet-wasm` or similar), once we decide on a JS reader — or precompute page-scoped JSON outputs in a Phase 5 step.
